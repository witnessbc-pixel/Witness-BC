Safety note (read first)

  • Do not upload sensitive personal data to shared folders or public links.
  • Do not edit originals: keep originals clean; store notes separately.
  • Privacy-first default: if publishing increases risk, publish the pattern and retain the raw record privately.
Boundary: education only — not legal advice. No case review. No outcomes sold.

Naming convention (minimum standard)

Format: YYYY-MM-DD__CATEGORY__SHORT-TITLE__SOURCEID.ext
Example: 2025-02-14__EMAIL__PAYMENT-CHANGE__SRC-017.pdf
Tip: keep titles short. The Source ID is what stays stable over time.

5-minute setup

  • Create a folder: WBC-RECORDS/
  • Inside: /ORIGINALS, /NOTES, /EXPORTS
  • Start an index file: source-index.txt
  • Assign Source IDs: SRC-001, SRC-002…
  • Back up in two places (local + cloud).

Source Index (example)

SRC-017 → 2025-02-14__EMAIL__PAYMENT-CHANGE__SRC-017.pdf (captured 2025-02-14; notes in NOTES/SRC-017.txt)
SRC-018 → 2025-02-16__WEB__POLICY-PAGE__SRC-018.pdf (URL archived; screenshot saved)
Goal: make “later verification” easy without hunting across random filenames.
